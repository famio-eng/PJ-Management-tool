# Claude Code 指示文 — PJ-Management-tool 改善実装

## 対象リポジトリ
https://github.com/famio-eng/PJ-Management-tool  
対象ファイル：`index.html`（単一ファイルにCSS・JS全内包）

---

## 前提・方針

- 既存の仕様書 `spec_v0.2.md` に基づいて実装済みのアプリに対して、下記の改善を追加する
- 既存の動作を壊さないよう、差分追加・修正の形で対応すること
- ライブラリはすでに読み込まれているD3.js v7・SheetJSを引き続き使用する
- データ永続化はLocalStorage（`pj_settings` / `pj_comments` / `pj_actions` / `pj_timeline_items`）を使用する

---

## ① ビュー共通（全画面に適用）

### 1-1. 「報告日」表示を「次回定例予定日」に変更
- サイドバーおよび各画面で「報告日」と表示しているラベルをすべて「**次回定例予定日**」に変更する
- 変数名・LocalStorageキー名はそのままで構わない（表示ラベルのみ変更）

### 1-2. 列幅の手動調整
- 各テーブルビュー（定例ビュー・機能別進捗報告・ガントチャートの固定列）で、列ヘッダーの境界をドラッグして列幅を変更できるようにする
- デフォルト列幅は「各列ヘッダー文字列の幅＋セル内の最大文字列幅」に自動フィットさせる（`ch`単位や`max-content`を活用）
- 調整した列幅はLocalStorage（キー：`pj_column_widths`）に画面ごとに保存し、次回起動時に復元する

### 1-3. 列フィルター機能
- 各テーブルの列ヘッダーにフィルターアイコン（▼）を追加する
- クリックするとドロップダウンが開き、その列に含まれる値の一覧をチェックボックスで表示する
- チェックを外した値を持つ行を非表示にする（DOMから削除ではなく`display:none`）
- 「すべて選択」チェックボックスを最上部に設置する
- フィルター状態はLocalStorage（キー：`pj_filters`）に画面ごとに保存し、次回起動時に復元する

### 1-4. 列の並び替え
- 各テーブルの列ヘッダーをドラッグ＆ドロップで並び替えできるようにする
- HTML5のdraggable属性を使用してシンプルに実装する
- 並び替えた順序はLocalStorage（キー：`pj_column_order`）に画面ごとに保存し、次回起動時に復元する

### 1-5. ナイトモード
- 右上のナビゲーションバーに🌙アイコンのトグルボタンを追加する
- クリックでダーク/ライトテーマを切り替える
- CSS変数（`--bg`, `--text`, `--border`, `--card-bg`等）をルートに定義し、`body.dark`クラスの付け外しで切り替える
- ダークモード時の配色例：背景 `#1a1a2e`、カード `#16213e`、テキスト `#e0e0e0`、ボーダー `#3a3a5c`
- 既存のステータス配色（On-going/Delay/Close/nActive/Disable）はダークモード時も視認性を保つよう輝度調整する
- 設定はLocalStorage（キー：`pj_theme`）に保存し、次回起動時に復元する

### 1-6. プロジェクト名の編集ウィンドウを削除
- サイドバー等にあるプロジェクト名をクリック編集できるUI（インライン編集・モーダル等）を削除する
- プロジェクト名は読み取り専用テキストとして表示するだけにする

## ② 大日程タイムライン（画面②）

### 2-1. 予測バーを実進捗バーに変更
- 現在「🟡 予測バー（開始日〜終了日）」として表示しているバーを廃止する
- 代わりに「**実進捗バー**（実績開始日〜実績終了日、未完了タスクは実績開始日〜TODAY）」をグレー（`#A0A0A0`）で基準計画バーに重畳表示する
- 実績開始日が空欄のタスクは実進捗バーを表示しない

### 2-2. 担当部門別バー色分け
- `リソース名`列を参照し、以下の部門カラーマップを定義してバーを色分けする
  ```
  'Pj-L'   : '#4472C4'  // 青
  'RA'     : '#ED7D31'  // オレンジ
  '工場'   : '#A9D18E'  // 緑
  'EL'     : '#FF0000'  // 赤
  'SW'     : '#9DC3E6'  // 水色
  'VV'     : '#7030A0'  // 紫
  'BRT'    : '#FFD966'  // 黄
  'PRM'    : '#F4B183'  // サーモン
  'PDA'    : '#00B0F0'  // シアン
  'Service': '#92D050'  // 黄緑
  'Market' : '#FF66CC'  // ピンク
  'IP'     : '#C9C9C9'  // 銀
  '複数部門': '#808080' // 共通グレー
  ```
- `リソース名`に複数部門がカンマ区切りで含まれる場合は「複数部門」色（`#808080`）を使用する
- 凡例をタイムライン右上または下部に表示する

### 2-3. 「＋アイテム追加」ボタンを非表示にする
- 「＋アイテム追加」ボタンをUIから削除（またはCSSで`display:none`）する
- 既存の登録データ・LocalStorageスキーマはそのまま維持する（将来復活を考慮）

## ③ NW図（画面③）

### 3-1. 完了タスクのグレーアウト
- ステータスが`Close`のタスクノードを、通常の配色ではなくグレー（背景`#D9D9D9`、文字`#888888`、枠線`#AAAAAA`）で表示する
- CPバッジ・赤枠も非表示にする（Closeタスクはクリティカルパス強調不要）

### 3-2. 担当機能名（リソース名）の表示
- 各タスクノードに、既存の「WBS番号・タスク名・ステータス・達成率」に加えて、**「担当：〇〇」**（`リソース名`の値）を表示する
- ノードの高さを若干拡張して収める（例：最小高さ `80px` → `100px`）

### 3-3. タスク詳細ウィンドウ（右パネル）のPhase 2表示予定
- 右パネルの現状空欄部分に、以下のプレースホルダーテキストを表示する：
  ```
  【Phase 2 追加予定】
  ・トータルフロート（TF）
  ・フリーフロート（FF）
  ・最早開始日（ES）/ 最早終了日（EF）
  ・最遅開始日（LS）/ 最遅終了日（LF）
  ```
- スタイルは薄いグレー文字（`#AAAAAA`）でイタリック表示し、「未実装」と分かるようにする

### 3-4. タスクのサブグラフ表示（ダブルクリック or 右クリック）
- CPモード・全体モードでタスクノードを**ダブルクリック**または**右クリック**した場合：
  - そのタスクを中心とした周辺サブグラフ（デフォルト：前後2階層）をNW図エリア下部にパネルを展開して表示する
  - 下部パネルはNW図メインエリアと同じD3.jsによるグラフ描画を使用する（既存のサブグラフモード描画ロジックを流用する）
  - 右クリック時はブラウザのコンテキストメニューを抑制（`event.preventDefault()`）し、「サブグラフを表示」のカスタムメニューを1項目表示する
  - 下部パネルには閉じるボタン（✕）を設置する
  - 階層数はスライダー（1〜5）でユーザーが変更できるようにする

## ④ ガントチャート（画面④）

### 4-1. タイトル変更
- 画面タイトルを「**ガントチャート（WBS）**」に変更する

### 4-2. 実績バーの重畳表示
- 現在「基準計画バー」と「実績バー」が2行に分かれている場合、**1行に重畳表示**する
  - 下層：🔵 基準計画バー（`#4472C4`、半透明 `opacity:0.6`）
  - 上層：🟢 実績バー（`#00B050`、`opacity:0.85`）を重ねて表示
- 実績開始日が空欄のタスクは実績バーを表示しない
- 未完了タスクの実績バーは「実績開始日〜TODAY」で表示する

### 4-3. 列ヘッダー項目名の追加
- 現在項目名が表示されていない固定列（ID・WBS番号・タスク名・担当部門・達成率・ステータス・NW図アイコン）に、仕様書記載の列名を正しく表示する
- ガント部分の列ヘッダーも「月」「日付（週開始日）」の2段ヘッダーを正しく表示する

### 4-4. サブグラフアイコンのポップアップ
- 各タスク行のNW図アイコンをクリックした際、新しいウィンドウ（`window.open`）またはモーダルダイアログでサブグラフを表示する
- モーダル方式を推奨：画面中央にオーバーレイ表示し、外側クリックまたは✕ボタンで閉じる
- モーダル内のNW図は③と同じD3.jsベースのサブグラフ描画を流用する

## ⑤ 定例ビュー（画面⑤）

### 5-1. 進捗コメントの別管理
- 定例ビューのコメント欄は`screen: "meeting"`として保存し、機能別進捗報告（`screen: "dept"`）のコメントとは**完全に別のデータ**として管理する
- LocalStorageキーを`pj_comments_meeting_{taskId}_{weekKey}`と`pj_comments_dept_{taskId}_{weekKey}`に分離する（または既存の`screen`フィールドで確実にフィルタリングする）
- 両画面で互いのコメントは表示しない

### 5-2. 期限列の削除
- 各セクションのテーブルから「期限（入力欄）」列を削除する
- アクションの期限は「＋アクション追加」機能で管理する

---

## ⑥ 機能別進捗報告（画面⑥）

### 6-1. 「All」タブの追加
- 既存の12部門タブの先頭（左端）に「**All**」タブを追加する
- Allタブでは全12部門のタスクを一覧表示する（部門列も表示して識別できるようにする）

### 6-2. コメントのAllタブ・各部門タブ間の同期
- コメントのLocalStorageキーは`taskId`と`weekKey`を基準にする（部門タブに依存しない）
- Allタブでコメントを入力・送信した場合、同じタスクの各部門タブにも同じコメントが表示される（逆も同様）
- コメントデータの取得・表示ロジックを`taskId + weekKey`のキーで統一し、どのタブから入力しても同じデータを参照するように修正する

### 6-3. 送信完了通知
- 「送信」ボタンを押した後、画面上部（またはボタン近傍）に**トーストメッセージ**を表示する
- 表示内容：「✅ コメントを保存しました（タスク名・担当：〇〇）」
- トーストは3秒後に自動的にフェードアウトする
- CSSアニメーション（`@keyframes`）で実装する

## 実装上の注意事項

1. **既存機能を壊さないこと**。特にステータス計算ロジック（`StatusEngine`）・データ読み込み（`DataLoader`）・NW図レイアウト（`NWGraphEngine`）は変更しないこと
2. 列幅調整・フィルター・列順序の3機能はLocalStorageへの保存タイミングを「変更直後」にし、ページリロード後に確実に復元されること
3. ナイトモードはCSSのカスケードで対応し、既存のインラインスタイルやステータス配色への影響を最小化すること
4. D3.jsの描画関数はナイトモード切り替え時に再描画するか、CSS変数で色を参照するよう修正すること
5. サブグラフ描画はNW図の既存ロジックを関数化・再利用すること（コードの重複を避ける）
6. 全変更後、ブラウザコンソールにエラーが出ないことを確認すること
