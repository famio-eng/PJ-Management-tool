### A-1. 全体共通：表示項目の編集（設定画面）

**要件**：
- どのビューでも表示する列・非表示にする列をユーザーが選択できるようにする
- 対象列候補：`ID / WBS番号 / アウトラインレベル / タスク名 / Note / 期間 / 開始日 / 終了日 / 基準開始日 / 基準終了日 / 実績開始日 / 実績終了日 / 残存期間 / 先行タスク / 後続タスク / リソース名 / 達成率 / 状況 / 成果物 / クリティカル / アクティブ / 進捗状況`
- 設定変更はナビゲーションバーの「⚙ 設定」ボタンから開くモーダル画面で行う
- モーダル内にタブを設け、「表示列」タブを設置する
- 各列名の横にチェックボックスを配置し、チェックON=表示・OFF=非表示とする
- デフォルトのON列：`タスク名 / リソース名 / 開始日 / 終了日 / 達成率 / 状況 / 進捗状況`
- 設定はLocalStorage（キー：`pj_visible_columns`）に保存し、次回起動時に復元する
- 設定変更は「適用」ボタン押下時に即時反映する

---

### A-2. 全体共通：担当部門のカンマ区切り対応

**要件**：
- `リソース名`列の値に `,` が含まれる場合（例：`RA,Pj-M`）、複数部門が割り当てられているものとして扱う
- **フィルター機能**：カンマ区切りの各部門を個別の値として列挙する
  - 例：`RA,Pj-M` の行は「RA」チェックON時も「Pj-M」チェックON時も表示される
  - フィルタードロップダウンの値一覧も「RA,Pj-M」という複合値ではなく「RA」「Pj-M」として個別表示する
- **色分け**：カンマ区切りの場合は既存の「複数部門」色（`#808080`）を適用する（既存実装と統一）
- **修正箇所**：フィルター生成時に以下のように分割処理を追加する
  ```javascript
  // 変更前
  values.add(row['リソース名']);
  // 変更後
  const resources = (row['リソース名'] || '').split(',').map(s => s.trim()).filter(Boolean);
  resources.forEach(r => values.add(r));
  ```
- フィルター適用時も同様に分割して判定する

### A-3. 全体共通：表記ゆれ正規化（エイリアス設定）

**要件**：
- Excelの`リソース名`列に入っている値が、アプリの部門カラーマップのキーと表記が異なる場合に正規化するマッピングを設定できるようにする
- 例：`Pj-M` → `Pj-L`、`MSA` → `Service`、`PJメンバー` → `Pj-L` など
- 設定はA-1の「⚙ 設定」モーダルに「エイリアス」タブを追加して行う
- UIは以下の形式：
  ```
  Excelの値        →  正規化後の名前
  [入力欄        ]    [入力欄       ]  [＋追加]
  Pj-M             →  Pj-L            [削除]
  MSA              →  Service         [削除]
  ```
- 「＋追加」ボタンで行を追加、「削除」で行を削除
- 設定はLocalStorage（キー：`pj_alias_map`）に保存する
- データ読み込み時（ExcelパースのDataLoader内）に、`リソース名`の値をエイリアスマップで置換してから処理する
- エイリアスはカンマ区切りの各部門名に個別に適用する

### A-4. Excel周り：プロジェクト選択→自動読み込み

**要件・背景**：
- 現状：ユーザーが毎回「ファイルを選択」ボタンでExcelをアップロードする必要がある
- 目標：アプリ起動時にプロジェクト名を選択するだけで、対応するExcelファイルが自動で読み込まれる

**運用前提**：
- ExcelファイルはSharePoint上の特定フォルダに格納される（Phase 2以降）
- Phase 1b現在はLocalStorage + 手動アップロードの組み合わせで対応する

**実装内容**：
1. アプリ起動時（またはサイドバー上部）に「プロジェクトを選択」ドロップダウンを表示する
2. ドロップダウンの選択肢は「＋新規プロジェクトを追加」から、プロジェクト名と対応するExcelファイルをセットで登録する
3. 登録内容はLocalStorage（キー：`pj_project_list`）に保存する
   ```json
   [
     { "name": "Improved ITM", "lastLoaded": "2026-05-12", "data": "<SheetJSのJSON文字列>" }
   ]
   ```
4. 登録済みプロジェクトを選択すると、保存済みのデータを即時読み込んで描画する（Excelファイルの再アップロード不要）
5. 「更新」ボタンを設け、クリックするとファイル選択ダイアログが開いて新しいExcelで上書き更新できる
6. 「削除」ボタンで登録済みプロジェクトを削除できる

**注意**：ExcelのJSONデータをLocalStorageに保存するため、ファイルサイズが大きい場合（5MB超）はエラーになる可能性がある。その場合はエラーメッセージで「ファイルが大きすぎます。直接読み込みモードを使用してください」と表示し、従来の直接アップロード方式にフォールバックする。

### A-5. ダッシュボードビュー：列の並び替え

**要件**：
- ダッシュボードの「遅延タスク一覧」「直近タスク一覧」テーブルの列ヘッダーをドラッグ＆ドロップで並び替えできるようにする
- 既存の`applyColReorder()`関数を流用する
- 並び順はLocalStorage（キー：`pj_column_order_dashboard`）に保存する

---

### A-6. ダッシュボードビュー：遅延タスクのフィルター・並び替え

**要件**：
- 「遅延タスク一覧」テーブルの列ヘッダーにフィルターアイコン（▼）と昇順/降順ソートアイコン（↑↓）を追加する
- フィルターは既存の`applyColFilter()`関数を流用する
- ソートはヘッダークリックで昇順→降順→解除の3ステートをローテーションする
- ソート状態はLocalStorageには保存しない（セッションごとにリセットでよい）

---

### A-7. タイムラインビュー：タスク一覧の列幅調整・表示項目編集

**要件**：
- タイムライン左側のタスク一覧パネルの列幅をドラッグで調整できるようにする
- 既存の`applyColResize()`関数を流用する
- 保存キー：`pj_column_widths_timeline`
- 表示する列もA-1の設定に連動させる（設定で非表示にした列はタイムラインのタスク一覧にも表示しない）

---

### A-8. タイムラインビュー：サマリータスクの折りたたみ

**要件**：
- `アウトラインレベル`が1のタスク（サマリータスク）の行に▶/▼トグルアイコンを表示する
- クリックすると配下の子タスク（アウトラインレベルが2以上で、直近の上位サマリータスクに属するもの）を折りたたむ/展開する
- 折りたたんだ場合はD3のタイムラインバーも非表示にする（行とバーを同期する）
- 折りたたみ状態はLocalStorage（キー：`pj_timeline_collapsed`）に保存し、次回起動時に復元する
- 判定ロジック：`アウトラインレベル`列の値を使って親子関係を構築する

---

### A-9. タイムラインビュー：タスク一覧の縦幅拡張

**要件**：
- 現在タスク一覧の表示縦幅が狭く、下部に余白が多い
- タスク一覧とD3タイムラインエリアが画面の残余高さをすべて使うよう、`height: calc(100vh - Xpx)` でコンテナの高さを設定する（Xはナビゲーションバー＋コントロールバーの合計高さ）
- リサイズ時も正しく追従するよう`ResizeObserver`または`window.onresize`で再計算する
