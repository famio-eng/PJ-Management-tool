### B-1. NW図ビュー：タイムラインNWモードの追加

**概要**：
既存のD3トポロジカルソートNW図（依存関係モード）はそのまま残しつつ、新たに「タイムラインNWモード」を追加する。モード切替ボタンでユーザーが選択できるようにする。

---

#### B-1-1. モード切替UIの追加

- NW図コントロールバーに「**🔗 依存関係モード**」「**📅 タイムラインNWモード**」のトグルボタンを追加する
- 既存の「CPモード」「全体モード」はそれぞれのモード内で引き続き動作する
- モード選択状態はLocalStorage（キー：`pj_nw_mode`）に保存し、次回起動時に復元する

---

#### B-1-2. タイムラインNWモードのレイアウト仕様

**基本構造**：
- SVGエリア上部に**タイムスケール（月単位、2段：年／月）**を固定表示する
- 各タスクノードの**X座標を`基準開始日`に対応**させてレイアウトする
- 各タスクノードの**Y座標はアウトラインレベル**（`アウトラインレベル`列の値）を基準に行割り当てする
  - 同レベルのタスクが重なる場合はY方向にずらして衝突を回避する
- ノード間の依存関係矢印は既存のD3 `link` 描画を流用する

**ノードの構成**（ノード幅 = 期間に比例、最小幅 120px）：
```
┌─────────────────────────────┐
│ WBS番号  タスク名            │  ← 上段：テキスト情報
│ 担当：RA  ● Delay  80%      │
├─────────────────────────────┤
│ ████████░░░░░░░░░░░░░░░░░░  │  ← 下段：基準計画バー（青）
│ ██████                      │       実績バー重畳（グレー）
└─────────────────────────────┘
```
- 上段：WBS番号・タスク名・担当部門・ステータス・達成率（既存ノード表示内容を流用）
- 下段：基準計画バー（`#4472C4`、`opacity:0.65`）の上に実績バー（`#A0A0A0`）を重畳
- ノード幅の計算：`(基準終了日 - 基準開始日) / 1日 × pxPerDay`
- `pxPerDay`（1日あたりのピクセル数）はデフォルト`3px`。後述のズームで変更可能

**タイムスケール**：
- 月単位の縦グリッド線（薄いグレー `opacity:0.3`）を全エリアに引く
- 上部ヘッダーに年・月ラベルを表示する

**横幅の制御（重要）**：
- デフォルト表示期間：プロジェクト全体の開始〜終了（全タスクの`基準開始日`最小値〜`基準終了日`最大値）
- コントロールバーに**表示期間フィルター**を設置する
  - 「全期間」「今後3ヶ月」「今後6ヶ月」「今後1年」のプリセットボタン
  - 表示期間外のノードは非表示（`display:none`）にする（削除しない）
- コントロールバーに**ズームスライダー**（`pxPerDay`：1〜10px、デフォルト3px）を設置する
  - スライダー操作時にSVGを再描画する
- 横スクロールは既存NW図と同様にD3のpan/zoomで対応する

---

#### B-1-3. 実装上の注意事項

- **既存のD3描画関数を破壊しないこと**。タイムラインNWモードは独立した描画関数（`renderTimelineNW()`）として実装し、モード切替時に既存の`renderNW()`と`renderTimelineNW()`を切り替えて呼び出す構造にする
- タイムラインモードでの右クリック・ダブルクリックサブグラフ表示（B-1の旧3-4）も引き続き動作させること
- `基準開始日`が`null`のタスクはタイムラインモードで非表示にする（依存関係モードは従来通り表示）
- ノードの衝突回避アルゴリズムはシンプルに：同じ月に開始するタスクが同じY行にある場合、後続をY方向に`ノード高さ + 10px`ずらす
- D3のpan/zoom（`d3.zoom()`）はタイムラインモードでも有効にする。ただしX方向のズームはスライダーで制御するため、`d3.zoom()`のscaleはY方向のみ、またはpanのみに制限することを検討する

### B-5. ガントチャートビュー：サマリータスク内のサマリータスクの折りたたみ

**要件**：
- 現状、最上位サマリータスクのみ折りたたみができ、サマリータスク内の子サマリータスクが折りたたみできない
- 修正内容：`アウトラインレベル`を再帰的に使い、任意の階層のサマリータスクで折りたたみを可能にする

### B-7. 定例ビュー：UIの改善

#### B-7-1. 進捗コメント記入欄のサイズ
- 進捗コメントの`<textarea>`の幅を、その列の現在の幅に合わせて自動伸縮させる（`width: 100%`）
- 高さは最低3行（`rows="3"`）とし、入力内容に応じて自動拡張する（`auto-resize`パターン）
  ```javascript
  textarea.addEventListener('input', function() {
    this.style.height = 'auto';
    this.style.height = this.scrollHeight + 'px';
  });
  ```

#### B-7-2. 縦スクロールバーのスタイル改善
- 細いデフォルトスクロールバーをガントチャートビューと同じスタイルに統一する

  ```css
  .meeting-container::-webkit-scrollbar { width: 10px; }
  .meeting-container::-webkit-scrollbar-track { background: var(--bg); }
  .meeting-container::-webkit-scrollbar-thumb { background: #888; border-radius: 5px; }
  .meeting-container::-webkit-scrollbar-thumb:hover { background: #555; }
  ```

#### B-7-3. 「期限」列の意図の明確化

## 実装上の注意事項

1. Session Aの`pj_visible_columns`設定はSession Bの各ビューでも参照すること
2. 折りたたみ（B-5・A-8）はD3描画エリアとHTMLテーブルの両方に影響する場合があるため、再描画トリガーを慎重に実装すること
3. スクロールバーのCSS（`::-webkit-scrollbar`）はChrome/Edge系のみ有効。Firefoxは`scrollbar-width: thin; scrollbar-color: #888 var(--bg);`を別途追加すること
4. B-4のTODAY線はD3描画のタイミング（`renderGantt()`）に合わせて描画すること。ビュー切り替え時に再描画されるよう既存のレンダリングフックに組み込むこと
