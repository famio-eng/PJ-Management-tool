## 未解決バグ・既知の問題

### [Bug-A] ガントチャート固定列の旧LocalStorageキー残留
**現象**: 旧キー `pj_column_widths_gantt` に巨大な列幅が残っているユーザー環境では、初回表示時に固定列が幅いっぱいに展開される可能性がある。  
**対処（コード上は修正済み）**: resizeKey を `pj_column_widths_gantt_fixed` に変更済み。旧キーのクリアが必要な場合は `enhanceGanttTable()` 冒頭に以下を追加する：

```javascript
localStorage.removeItem('pj_column_widths_gantt'); // 旧キー強制削除
```

## 将来改良候補

### [Future-G1] ガントチャート横スクロールバーの配置
**要望**: 横スクロールバーをタブ（Excelインポートタブ等）と同じ高さに配置したい。現状は表示領域の下段に空白が生じている。  
**背景**: `max-height: calc(100vh - 140px)` でパネル高さを固定しているため、コンテンツがパネル高さより短い場合に下部余白が発生し、横スクロールバーが画面下端に張り付かない。  
**検討案**:
- `.gantt-split-outer` の高さをコンテンツに合わせる（`height: fit-content` 等）と、スクロールバーが常にコンテンツ直下に来る
- ただしウィンドウ高さを超えるコンテンツでは縦スクロールが壊れるため、`min-height + max-height` の組み合わせか、スクロールコンテナ設計の見直しが必要
- 代替案：横スクロールバーを `position: fixed; bottom: 0` で別要素として実装し、右パネルの scroll イベントと連動させる
