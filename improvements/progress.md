## 未解決バグ（要対応）

### [Bug-A] ガントチャート固定列がウィンドウ幅いっぱいに展開される
**現象**: ガントチャートビュー表示時、固定列（左パネル）がウィンドウ幅全体に広がり、タスク情報もカレンダーも確認できない。  
**原因（特定済み）**: `enhanceGanttTable()` の `resizeKey` が古い単一テーブル実装時のキー `'pj_column_widths_gantt'` を参照していた。このキーに保存されていた巨大な列幅がそのまま固定テーブルに適用されていた。  
**対処（コード上は修正済み・push済み）**: `resizeKey` を `'pj_column_widths_gantt_fixed'` に変更。しかし **ユーザー環境のLocalStorageには旧キーのデータが残っており**、ブラウザ側でキャッシュクリアまたは `localStorage.removeItem('pj_column_widths_gantt')` を実行しないと改善されない可能性が高い。  
**次のアクション**: ブラウザの開発者ツール（Application → Local Storage）で `pj_column_widths_gantt` キーを手動削除するか、`enhanceGanttTable()` 冒頭に旧キー削除コードを追加する。

```javascript
// enhanceGanttTable() 先頭に追加すべきコード
localStorage.removeItem('pj_column_widths_gantt'); // 旧キー強制削除
```

### [Bug-B] NW図サブグラフが新規ウィンドウで描画されない
**現象**: サブグラフウィンドウは開くが、深さスライダーとズームリセットボタンのみ表示。D3グラフが描画されない。  
**原因（特定済み・複数）**:
1. 旧実装: `document.write()` → ChromeがパーサーブロッキングポリシーでCDN `<script>` をブロック
2. 旧実装: 関数を IIFE `(function(){...})()` 内に定義 → `onclick="draw()"` 等のHTML属性からアクセス不可（`ReferenceError`）
3. Blob URLアプローチに切り替え後も描画されない → `window.opener._nwSubData` の取得タイミング問題の可能性、またはD3のロード完了前に `draw()` が実行されている可能性

**現在の実装**: Blob URL + CDN D3 `<script src>` in `<head>` + トップレベル関数  
**未確認事項**:
- `window.opener` が `null` になっていないか（ブラウザのpopup opener制限）
- D3がロードされる前に `draw()` が呼ばれていないか（`DOMContentLoaded` または `d3` の存在チェックが必要な可能性）
- Blob URLページでCDN読み込み自体がCSP等でブロックされていないか

**推奨アプローチ（未実装）**: D3をインライン埋め込み（minified d3をBase64またはそのままHTML文字列に含める）することでCDN依存を排除する。または `DOMContentLoaded` + `typeof d3 !== 'undefined'` チェックでロード順序を保証する。
