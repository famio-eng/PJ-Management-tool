## 1. アプリ概要

### 目的
医療機器製品開発プロジェクトのPMが、週次定例前の進捗把握・報告収集・定例運営を効率化するためのWebアプリ。MS Projectで管理しているWBSデータをExcelエクスポートし、ブラウザで視覚的に確認・報告入力できるようにする。

### 対象ユーザー
| ロール | 人数 | 主な使い方 |
|--------|------|-----------|
| PM（プロジェクトマネージャー） | 1名 | 全体確認・定例資料準備・アプリ管理 |
| 機能部門担当者 | 12部門（複数名） | 自部門タスクの進捗コメント記入 |

### 利用シナリオ（週次）
※定例の開催曜日はプロジェクトによって異なる。

1. **定例前日午後**：PMがMS ProjectをExcelエクスポート → OneDriveのdata/フォルダに上書き保存
2. **定例前日夜まで**：各担当者がブラウザでアプリを開き、自部門タスクに進捗コメントを記入・送信
3. **定例当日**：PMが定例ビューを開き、Teamsの画面共有で進捗確認・議論。必要に応じてブラウザ印刷でPDF出力

### 将来拡張（Phase 2）
- 複数PJ対応（PJセレクターで切替）。他PJの閲覧はPMが権限操作
- Microsoft Graph API連携によるコメント自動保存
- Power Automateによる定例前リマインダー通知（Teamsメッセージ）
- M365ログイン連携による担当者名自動識別
- NW図へのCPM要素追加（トータルフロート・フリーフロート・ES/EF/LS/LF）
- 定例アクションのToDoアプリ連携通知

## 3. 機能仕様

### 3.1 共通機能

#### データ読み込み
- 起動時にSharePoint/OneDrive上の`data/WBS_export.xlsx`を自動読み込み
- 読み込み失敗時はエラーメッセージを表示し、手動ファイル選択にフォールバック
- **状況報告日** = 次の定例予定日。PMが画面上から手動設定可能（LocalStorageに保存）

#### ステータス自動計算ロジック
MS Projectの計算式をJSで完全再現。`はい/いいえ`形式に対応。

```
【サマリータスク（アウトラインフィールド = "はい"）】
if アクティブ = "いいえ"                          → Disable
if 実績開始日 = null AND 報告日 < 基準開始日       → nActive
if 子タスクに Delay が1件以上                      → Delay
if 達成率 = 100 AND 残存期間 = 0                   → Close
else                                               → On-going

【通常タスク（アウトラインフィールド = "いいえ"）】
if アクティブ = "いいえ"                          → Disable
if 報告日 < 基準開始日                             → nActive
if 達成率 = 100                                    → Close
if 報告日 > 基準終了日                             → Delay
if 達成率 ≥ floor((報告日-基準開始日)/(基準終了日-基準開始日) × 100)
                                                   → On-going
else                                               → Delay
```

#### ステータス配色
| ステータス | 文字色 | 背景色 |
|-----------|--------|--------|
| On-going | #00B050 | #E2EFDA |
| Delay    | #FF0000 | #FFE0E0 |
| Close    | #0070C0 | #DDEEFF |
| nActive  | #808080 | #F2F2F2 |
| Disable  | #BFBFBF | #F2F2F2 |

#### 共通ナビゲーション
- 左サイドバーに6画面へのリンク
- 現在の状況報告日・PJ名を常時表示
- 各画面からNW図サブグラフへのジャンプ機能

---

### 3.2 画面①：ダッシュボード

**目的**：プロジェクト全体の健康状態を一目で把握

**表示要素**：
- KPIカード4枚（On-going件数 / Delay件数 / Close件数 / nActive件数）
- 遅延タスク一覧（WBS番号・タスク名・担当・基準終了日・達成率・CPフラグ・NW図アイコン）
- 今後2週間以内に開始すべきタスク一覧
- 今後2週間以内に完了予定のタスク一覧
- 機能部門別ステータス集計テーブル（行：12部門 × 列：各ステータス件数）

**インタラクション**：
- タスク行のNW図アイコンをクリック → 画面③NW図にジャンプし、そのタスクの周辺サブグラフを表示

---

### 3.3 画面②：大日程タイムライン

**目的**：プロジェクト全体のフェーズ進行とマイルストーンをMS Projectのタイムラインビューに近い形で確認

**表示要素**：
- 横軸：月単位（表示期間：6ヶ月 / 12ヶ月 / 18ヶ月 切替可）
- **フェーズバー（2本重ね）**：
  - 🔵 基準計画バー（基準開始日〜基準終了日）
  - 🟡 予測バー（開始日〜終了日）を重ねて表示
- **マイルストーン**：期間="0 days"のタスクをひし形◆で表示
- TODAY縦線（オレンジ）
- 各バー・◆にホバーでツールチップ表示（タスク名・日付・ステータス）

**手動登録機能**：
- 「＋アイテム追加」ボタンでフェーズバーまたはマイルストーンを手動登録
  - 入力項目：表示名・種類（フェーズ/マイルストーン）・日付または期間・表示色
- 登録データはLocalStorageに保存（Excelファイル更新後も維持）
- 登録済みアイテムは編集・削除可能
- 自動抽出アイテム（MSPデータ由来）と手動登録アイテムを視覚的に区別

---

### 3.4 画面③：NW図（タスク依存関係）

**目的**：先行タスク・後続タスクの依存関係とクリティカルパスを視覚的に把握

**表示モード（切替可）**：
1. **CPモード**（デフォルト）：クリティカルパスタスクのみ表示
2. **全体モード**：全タスクを表示
3. **サブグラフモード**：選択タスクの周辺N階層を表示（Nはユーザーが1〜5で設定可能）

**レイアウト**：左→右の階層レイアウト（トポロジカルソートに基づく列配置）

**ノード表示**：
- WBS番号・タスク名・ステータス・達成率%
- クリティカルパスノードは赤枠＋右上に「CP」バッジ
- ステータスに応じた背景色（3.1の配色表に準拠）
- 選択中ノードは強調表示（太枠）

**エッジ表示**：
- CP間の依存：赤矢印（太線）
- 通常の依存：青矢印（細線）

**右パネル（タスク詳細）**：
- タスク名・WBS・担当部門・基準開始/終了・達成率・ステータス・Note
- 「周辺NWを表示」ボタン → サブグラフモードに切替
- Phase 2：トータルフロート・フリーフロート・ES/EF/LS/LFを追加表示

**インタラクション**：
- タスクノードクリック → 右パネルに詳細表示
- ドラッグでパン、マウスホイールでズーム
- 「ズームリセット」ボタン
- 担当部門フィルタ（ドロップダウン）

---

### 3.5 画面④：WBSガントチャート（中・小日程）

**目的**：全タスクの基準計画・実績を週単位で確認

**固定列**：
ID / WBS番号 / タスク名（アウトラインレベルに応じてインデント）/ 担当部門 / 達成率 / ステータス / NW図アイコン

**ガント列**：
- 表示週数切替：2週 / 4週 / 13週 / 26週 / 52週
- 週単位の列、列ヘッダーに月・日付表示
- **2本バー（1行に重ねて表示）**：
  - 🔵 基準計画バー（基準開始日〜基準終了日）
  - 🟢 実績バー（実績開始日〜実績終了日、未完了はTODAYまで）
- クリティカルパスタスクの行は赤枠で強調
- TODAY列はオレンジ背景

**インタラクション**：
- NW図アイコンクリック → 画面③NW図のサブグラフにジャンプ
- サマリー行クリック → 配下タスクの折りたたみ/展開
- 表示週数の切替

---

### 3.6 画面⑤：定例ビュー

**目的**：週次定例で使う資料を自動生成。Teams画面共有・ブラウザ印刷に対応

**セクション構成**（自動抽出）：
1. 🔴 完了すべきだが未完了のタスク（ステータス = Delay）
2. 🟡 今後2週間以内に開始すべきタスク（ステータス = nActive かつ 基準開始日 ≤ 報告日+14日）
3. 🟢 今後2週間以内に完了予定のタスク（ステータス = On-going かつ 基準終了日 ≤ 報告日+14日）
4. 🔴 クリティカルパス上の進行中タスク（CP = はい かつ ステータス ∈ {On-going, Delay}）

**各セクションの列**：
WBS / タスク名 / 担当 / 基準終了 / 予測終了 / 達成率 / ステータス / 進捗コメント（入力欄）/ 期限（入力欄）

**アクション記録機能**：
- 定例中に「＋アクション追加」ボタンで宿題を記録
- 入力項目：内容・担当者・期限・優先度（高/中/低）
- 記録したアクションは画面下部に一覧表示
- LocalStorageに保存（次回起動時も残る）
- Phase 2：M365 ToDoアプリへの連携通知

**進捗コメント入力**：
- Phase 1：ブラウザ上で入力 → 「送信」ボタンで LocalStorageに保存
- Phase 2：Graph API経由で`reports.xlsx`に自動保存

**PDF出力**：
- 「印刷」ボタン → ブラウザ印刷ダイアログ
- 印刷用CSS：サイドバー・ボタン類を非表示、A3横向き推奨

---

### 3.7 画面⑥：機能別進捗報告

**目的**：各部門担当者が自分のタスクだけを確認し、進捗コメントを記入する

**部門タブ**：
- 画面上部に12部門（Pj-L / RA / 工場 / EL / SW / VV / BRT / PRM / PDA / Service / Market / IP）のタブ
- タブ切替で担当タスク一覧が切り替わる
- リソース名に複数部門が含まれる場合（例：`EL, SW`）は両方のタブに表示

**表示列**：
WBS / タスク名 / 基準開始 / 基準終了 / 達成率 / 予測開始 / 予測終了 / ステータス / 進捗コメント（入力欄）/ 担当者名（入力欄）

**コメント入力・保存**：
- コメント入力後「送信」ボタンで確定・LocalStorageに保存
- 送信済みコメントは背景色を変えて識別

**過去報告の表示**：
- 直近3週分の報告を右側に列追加して時系列表示（右スクロール）

**担当者名**：
- Phase 1：自由入力（初回入力後LocalStorageに記憶・次回自動補完）
- Phase 2：M365ログイン連携で自動設定

## 5. 技術実装メモ（Claude Code向け）

### ファイル構成
```
index.html   （単一ファイル。CSS・JS全て内包）
```

### 主要ライブラリ（全てCDN）
```html
<script src="https://cdnjs.cloudflare.com/ajax/libs/d3/7.8.5/d3.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/xlsx/0.18.5/xlsx.full.min.js"></script>
```

### モジュール構成（JS内部）
```
DataLoader        Excelファイルの読み込み・パース・列マッピング
StatusEngine      ステータス計算ロジック（MSP計算式の完全再現）
NWGraphEngine     先行/後続タスクのパース・トポロジカルソート・レイアウト計算
TimelineRenderer  D3.jsによる大日程タイムライン描画
NWRenderer        D3.jsによるNW図描画（ズーム・パン・クリック）
GanttRenderer     ガントチャートのHTMLテーブル生成
MeetingRenderer   定例ビューのHTML生成
DeptRenderer      機能別報告シートのHTML生成
StorageManager    LocalStorageの読み書き・スキーマ管理
UIController      ナビゲーション・画面切替・共通UI
```

### NW図レイアウトアルゴリズム
1. 先行タスク列をパースしてエッジリストを生成
2. トポロジカルソートで列（depth）を決定
3. 同一列内のノードを縦に均等配置
4. D3.jsのzoom/drag で操作

### SharePointからのExcel読み込み
```javascript
// 同一テナントの場合、CORSなしで直接fetchが可能
const response = await fetch(EXCEL_URL);
const arrayBuffer = await response.arrayBuffer();
const workbook = XLSX.read(arrayBuffer, {type: 'array', cellDates: true});
```

## 6. スコープ外（対応しないこと）

- MS Projectファイルの直接編集
- 基準日の自動更新（別途ExcelマクロV2で対応）
- ガントチャートのドラッグによる日程編集
- 外部システム（Jira・Redmineなど）との連携
- スマートフォン最適化
- オフライン動作（SharePointへの接続が必須）
